MPI_INIT

  • Fortran 95
  • Python

COMMAND USAGE: call MPI_INIT(ierr)

ierr is an integer that contains the error code.

COMMAND USAGE: MPI.Init()

MPI.Init() is called when MPI is imported from the mpi4py package. No need to initialize in python.

MPI COMM SIZE and RANK

  • MPI_COMM_SIZE returns the number of processors available.
  • MPI_COMM_RANK returns the current processor number.

  • Fortran 95
  • Python

COMMAND USAGE: call MPI_COMM_SIZE( comm, nproc, ierr )

COMMAND USAGE: call MPI_COMM_RANK( comm, iproc, ierr )

  • comm: Communicator that is holding all the processes.
  • nproc: INTEGER that will hold the number of processors.
  • iproc: INTEGER that will hold the current processor identifier.
  • ierr: Holds the error code, if any.

COMMAND USAGE: comm.size

COMMAND USAGE: comm.Get_rank()

  • comm.size: returns the number of processors being used.
  • comm.Get_rank(): returns the current processor identifier.

Sending data

Data needs to be sent from the root_processor, usually rank 0 processor, to the other processors.

  • Fortran 95
  • Python

COMMAND USAGE: CALL MPI_SEND( buffer, buffer size, MPI_TYPE, processor, tag, comm, ierr )

  • buffer: variable/array that will be operated on.
  • buffer size: size of buffer. number of elements being sent.
  • MPI_TYPE: data type that the buffers hold.
  • processor: processor number the data is being sent to.
  • tag: integer that keeps track of the message being sent. Useful if sending many messages.
  • comm: Communicator that is holding all the processes.
  • ierr: Holds the error code, if any.

COMMAND USAGE: comm.send( size 1 buffer, dest=destination )

COMMAND USAGE: comm.Send( buffer, dest=destination )

  • size 1 buffer: single element, float or integer.
  • dest=destination: destination is the processor number that will receive the data.
  • buffer: array that is being sent. Must send the length first with comm.send( ... )

{% comment %} Start of 'receive' section. {% endcomment %}

Receiving data

Data needs to be received from the root_processor, usually rank 0 processor, by the other processors.

  • Fortran 95
  • Python

COMMAND USAGE: CALL MPI_RECV( buffer, size, MPI_TYPE, source, tag, comm, status, ierr )

  • buffer: data that is being sent. Can be a single number or an array of any size.
  • size: number of elements that the buffer contains.
  • MPI_TYPE: data type that the buffers hold.
  • source: processor number that is sending the data.
  • tag: integer that keeps track of the message being sent. Useful if sending many messages.
  • comm: Communicator that is holding all the processes.
  • status: holds the status of the sending/receiving.
  • ierr: Holds the error code, if any.

COMMAND USAGE: size 1 buffer = comm.recv( source=processor )

COMMAND USAGE: comm.Recv( buffer, source=processor )

  • size 1 buffer: single element, float or integer.
  • source=processor: processor is the integer value of the processor that is sending the data.
  • buffer: use comm.Recv( ... ) if sending an array. The array has to be initialized beforehand.

Broadcasting data

Data is sent to all other processors, while the processor sending the data keeps a copy of the data.

  • Fortran 95
  • Python

COMMAND USAGE: call MPI_BCAST( buffer, buffer size, MPI_TYPE, root, comm, ierr )

  • buffer: data that is being sent. Can be a single number or an array of any size.
  • buffer size: size of buffer, number of elements being sent.
  • comm: Communicator that is holding all the processes.
  • nproc: INTEGER that will hold the number of processors.
  • iproc: INTEGER that will hold the current processor identifier.
  • ierr: Holds the error code, if any.

COMMAND USAGE: buffer_out = comm.bcast( buffer_in, root=0 )

  • comm = MPI.COMM_WORLD
  • buffer_in/out: data that is being sent/received. Can be a single number or an array of any size.
  • root: The number of the main processor.

Reducing data

Application of operation on a variable, if single variable, or on matching indicies across processors, if array.

  • Fortran 95
  • Python

COMMAND USAGE: CALL MPI_REDUCE( buffer, second buffer, size, MPI_TYPE, MPI_CALC, root, comm, ierr )

  • buffer: variable/array that will be operated on.
  • second buffer: buffer that will hold the final value.
  • size: size of buffer. second buffer must be the same size.
  • MPI_TYPE: data type that the buffers hold.
  • MPI_CALC: calculation that will be applied to each element.
  • root: root processor integer value.
  • comm: Communicator that is holding all the processes.
  • ierr: Holds the error code, if any.

COMMAND USAGE: second buffer = comm.reduce( buffer, op=operation, root=0 )

  • second buffer: buffer that will hold the final value.
  • buffer: variable/array that will be operated on.
  • operation: operation that will be applied to each element.
  • root: root processor integer value, usually 0.

MPI_Types

  • INTEGER: MPI_INTEGER
  • REAL: MPI_REAL
  • DOUBLE PRECISION: MPI_DOUBLE_PRECISION
  • COMPLEX: MPI_COMPLEX
  • LOGICAL: MPI_LOGICAL
  • CHARACTER: MPI_CHARACTER
  • MPI_PACKED

MPI_OPERATIONS

  • MPI_MAX: Finds maximum across all processors for each array index
  • MPI_MIN: Finds minimum across all processors for each array index
  • MPI_PROD: Finds product across all processors for each array index
  • MPI_SUM: Finds sum across all processors for each array index
  • MPI_LAN: Logical AND
  • MPI_LOR: Logical OR
  • MPI_LXOR: Logical Exclusive OR
  • MPI_MAXLOC: Maximum value and location
  • MPI_MINLOC: Minimum value and location
